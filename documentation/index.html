General usage

RTByte commands are always prefaced by a dash (-) on a server with default configuration. If this does not work, please contact the server staff of the Discord server you're trying to execute it on, as they may have changed it manually.

Parts of commands with [ ] around them are optional, whereas parts with < > are required.

User commands

COMMAND DESCRIPTION ALIASES
-8ball <question> Magic 8-ball, does exactly what the toy does, memes included. mirror, magicconch
-call Generates a video chat link for use in servers. videolink, screenshare, camera, videochat
-choice <choices>, <choices> Makes a decision based off of the choices given, no matter how many you include. choose, decide
-coinflip Flips a coin. ðŸ™‚ for heads, ðŸ™ƒ for tails. coin
-discordstatus Checks Discord's status and lets you know if there are any ongoing incidents. dstatus
-enlarge <emoji> Enlarges a specified emoji. Standard emojis will unfortunately not work. bigemoji
-help [command] Displays a list of commands, or extended help for a specified command. commands, cmd, cmds
-info Provides bot information, such as a list of authors and libraries. details, what
-invite Displays the link to invite RTByte to your server.
-joindate [user] Displays your account creation date along with the date you joined the current server you're on. Other users can be specified to fetch their account creation and server join dates. me, myself, age
-lyrics <song> Fetches lyrics and information from Genius for you. songlyrics, lyric
-ping Runs a connection test to Discord.
-quote <message ID> [channel] Quotes a message by ID. Mention the channel if the message you want to quote is not the same as the one you're sending the message in. q
-reminder <when> <message> [-dm] Sets a reminder for you, append -dm to have the reminder sent to you via DM. remindme, remind, setreminder
-roles <list|join|leave|add|remove> [user] [role] A series of commands to allow you to join roles by yourself. roleme, team, squad, role
-star [top|random] Fetch a random message from the starboard. Append top to get the top starred message instead. starred, stars
-weather <location> Fetch the current weather for a specified location. climate

Moderation commands

COMMAND DESCRIPTION ALIASES
-userinfo [user] Get information on a specified user. If no user is specified, it returns your info. uinfo
-channelinfo [channel] Get information on a specified channel. If no channel is specified, it returns your current channel. cinfo
-roleinfo [role] Get information on a specified role. If no role is specified, it returns your highest role. rinfo
-serverinfo Displays server information. sinfo, guildinfo
-sendmsg <channel/user> <message> Sends a message to the specified channel or user using RTByte. sendmessage, message, msg, echo
-modhistory [user] [case ID] Lists moderation actions taken against a user in the past. Refers to a specific case if a case ID is provided. history, case
-purge [user] <amount> [-all] [-s] Removes X amount of messages, optionally sent by Y user. Append -all to ignore the role hierarchy. p
-warn <user> [reason] [-s] Warns a specified user and logs the reason. w
-mute <user> <when> [reason] [-s] Mutes a specified user and logs the reason. m, tm, timeout
-unmute <user> Unmutes a specified user. um, untimeout
-kick <user> [reason] [-s] Kicks a specified user and logs the reason. k
-softban <user> [reason] [-s] Softbans a specified user and logs the reason. sb
-ban <user> [when] [reason] [-s] Bans a specified user and logs the reason. b
-vckick <user> [reason] [-s] Kicks a specified user from voice chat and logs the reason. vk, vck
-vcban <user> <when> [reason] [-s] Bans a specified user from voice chat and logs the reason. vb, vcb
-vcunban <user> Unbans a specified user from voice chat. vub, vcub

Admin commands

COMMAND DESCRIPTION ALIASES
-conf <set|show|remove|reset> (key) (value) Define per-server configuration via commmand.
-createcmd <command name> <command response> Creates a custom command for the server. addcmd, newcmd, createcommand, addcommand
-createcmd <command name> <command response> Creates a custom command for the server. addcmd, newcmd, createcommand, addcommand
-deletecmd <command name> Deletes a custom command from the server. removecmd, delcmd, deletecommand, removecommand
-updatecmd <command name> <new command response> Updates an existing custom command in the server. updcmd, changecmd, updatecommand, changecommand, editcmd
